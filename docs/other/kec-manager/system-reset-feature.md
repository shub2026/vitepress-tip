# 系统设置 - 重置系统功能说明

## 功能概述#

在系统设置页面中新增了"重置系统"功能模块，支持分别清空各项基础数据、培养方案和系统设置，所有操作均配备二次确认机制和智能级联清空策略，确保数据完整性。

## 功能特性#

### 1. 精细化数据清空#

基础数据清空细分为6个独立操作，每个操作都可以单独执行：
- **清空专业** - 删除所有专业信息
- **清空学院** - 删除所有学院信息
- **清空层次** - 删除所有学历层次信息
- **清空课程** - 删除所有课程信息
- **清空教材** - 删除所有教材信息
- **清空班级** - 删除所有班级信息

### 2. 培养方案与设置清空#

- **清空培养方案** - 删除所有培养方案及相关关联数据
- **清空系统设置** - 删除所有系统配置项

### 3. 二次确认机制#

每个清空操作都需要经过严格的二次确认：
- 输入特定的确认文字（如"确认清空"）
- 显示详细的操作影响说明
- 提示级联影响范围
- 只有确认文字完全匹配才能执行操作

### 4. 智能级联清空策略#

根据数据依赖关系，采用三层级联清空策略：

#### Level 0 - 基础数据层#
- 专业 (Major)
- 学院 (College)
- 层次 (Level)
- 课程 (Course)
- 教材 (Textbook)
- 班级 (Class)

#### Level 1 - 培养方案层#
- 培养方案主表 (TrainingPlan)
- 培养方案课程 (PlanCourse)
- 培养方案学期 (PlanCourseSemester)
- 培养方案教材 (PlanTextbook)

#### 级联规则#

**课程/教材清空时：**
1. 先清空 Level 1 培养方案相关数据
2. 再清空目标基础数据

**专业/学院/层次清空时：**
1. 检查是否存在班级数据
2. 如有班级数据，阻止操作并提示用户先清空班级
3. 如无班级数据，执行级联清空培养方案和目标数据

**班级清空时：**
- 独立操作，无级联影响

### 5. 清空顺序说明#

为确保操作顺利，建议按以下顺序执行清空：

```
班级 → 专业/学院/层次 → 课程/教材 → 培养方案 → 系统设置
```

**注意事项：**
- 清空专业、学院、层次前，必须先清空班级
- 清空课程、教材会自动清空相关的培养方案数据
- 培养方案清空不会影响基础数据
- 系统设置清空是独立操作

### 6. 视觉区分设计#

不同清空按钮采用不同的颜色样式进行区分：

| 数据类型 | 按钮类型 | 颜色标识 |
|---------|---------|---------|
| 专业 | danger | 红色 |
| 学院 | danger | 红色 |
| 层次 | danger | 红色 |
| 课程 | warning | 橙色 |
| 教材 | warning | 橙色 |
| 班级 | info | 蓝色 |
| 培养方案 | danger | 红色 |
| 系统设置 | default | 灰色 |

### 7. 左右分栏布局#

重置系统功能采用左右分栏设计（各占50%）：

**左侧 - 操作区域：**
- 8个清空按钮分组排列
- 基础数据区（6个按钮）
- 培养方案区（1个按钮）
- 系统设置区（1个按钮）

**右侧 - 提示面板：**
- 重要提醒：强调数据不可恢复
- 前置条件：说明清空依赖关系
- 清空顺序：提供推荐操作顺序。
- 颜色说明：解释按钮颜色含义
- 级联影响：说明智能级联清空机制。

### 后端实现#

**文件：** `server/src/routes/settings.routes.js`

**API端点：**

| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/settings/reset/majors` | POST | 清空专业（含班级依赖检查） |
| `/api/settings/reset/colleges` | POST | 清空学院（含班级依赖检查） |
| `/api/settings/reset/levels` | POST | 清空层次（含班级依赖检查） |
| `/api/settings/reset/courses` | POST | 清空课程（级联清空培养方案） |
| `/api/settings/reset/textbooks` | POST | 清空教材（级联清空培养方案） |
| `/api/settings/reset/classes` | POST | 清空班级（独立操作） |
| `/api/settings/reset/plans` | POST | 清空培养方案 |
| `/api/settings/reset/settings` | POST | 清空系统设置 |

**智能级联逻辑示例：**

```javascript
// 清空专业 - 检查班级依赖
router.post('/reset/majors', async (req, res, next) => {
  try {
    const classCount = await prisma.class.count();
    if (classCount > 0) {
      return fail(res, '系统中存在班级数据，请先清空班级后再清空专业');
    }
    
    // 级联清空培养方案相关数据
    await prisma.planTextbook.deleteMany();
    await prisma.planCourseSemester.deleteMany();
    await prisma.planCourse.deleteMany();
    await prisma.trainingPlan.deleteMany();
    await prisma.major.deleteMany();
    
    success(res, null, '专业数据已清空（已级联清空相关的培养方案）');
  } catch (e) { 
    next(e); 
  }
});
```

## 数据完整性保障#

### 外键约束处理#

Prisma Schema中的关键约束：
- `Class.majorId` - 必需字段（非空）
- `Class.collegeId` - 必需字段（非空）
- `Class.levelId` - 必需字段（非空）

### 依赖关系图#

```
培养方案 (TrainingPlan)
  ├─ 培养方案课程 (PlanCourse)
  │   └─ 培养方案学期 (PlanCourseSemester)
  └─ 培养方案教材 (PlanTextbook)
       └─ 课程 (Course)
       └─ 教材 (Textbook)

班级 (Class)
  ├─ 专业 (Major) ← 必需
  ├─ 学院 (College) ← 必需
  └─ 层次 (Level) ← 必需
```

### 清空策略#

1. **自上而下原则**：先清空上层应用数据，再清空底层基础数据
2. **依赖检查原则**：清空被引用数据前，检查是否存在引用记录
3. **级联清理原则**：清空基础数据时，自动清理相关的派生数据

## 用户体验优化#

### 操作确认流程#

1. 用户点击清空按钮
2. 弹出确认对话框
3. 显示操作影响和级联提示
4. 要求输入确认文字
5. 验证通过后执行清空
6. 显示成功消息和实际删除数量

### 错误提示#

- **依赖冲突**：清晰说明需要先清空的数据
- **验证失败**：提示确认文字不匹配
- **操作失败**：显示具体错误信息

### 成功反馈#

- 显示删除的记录数量
- 说明级联清空的范围
- 自动刷新相关列表

## 注意事项#

⚠️ **重要提醒：**
- 所有清空操作不可恢复，请谨慎操作
- 建议在清空前备份重要数据
- 清空操作会立即生效，无法撤销

🔒 **前置条件：**
- 清空专业、学院、层次前，必须先清空班级
- 系统会检查依赖关系并给出明确提示

🎨 **颜色说明：**
- 🔴 红色按钮：高危操作（专业、学院、层次、培养方案）
- 🟠 橙色按钮：中等风险（课程、教材）
- 🔵 蓝色按钮：低风险（班级）
- ⚪ 灰色按钮：系统设置
