# Vitepress构建基本命令
## 拉拉取文件
进入当前文件夹后拉取项目文件，适用于本地过或服务器执行
```
# HTTP方式
 git clone https://gitee.com/shub77/Vitepress-tip
 git clone https://github.com/shub2026/Vitepress-tip

# SSH连接
 git remote add origin git@gitee.com:shub77/vitepress-tip.git

```

## 环境和构建
确认本地已经安装Node.js
进入项目文件夹，安装NPM服务，启动开发环境，已经安装了就直接构建（预览本地构建服务）
构建静态文件

```
# npm安装
npm install

# 启动开发服务器
npm run docs:dev
 
# 构建静态
npm run docs:build

# 预览生产构建
npm run docs:preview
```
**预览生产构建**
如果已经执行过npm run docs:build生成了静态文件，可以使用以下命令本地预览生产版本。
这会启动一个本地静态Web服务（默认端口4173），以.vitepress/dist作为源文件。

## 提交代码到 GitHub
1. 打开 VS Code 的终端（Ctrl + ~）
2. 依次运行这些命令：
```
# 暂存所有改动
git add .

# 创建一个提交
git commit -m "部署到 GitHub Pages"

# 上传到 GitHub
git push
```
## 项目结构速查

```
项目文件夹/
├── docs/                    # 网站内容（这是重点！）
│   ├── index.md            # 首页
│   ├── content_A/          # 内容分类 A
│   │   ├── A1.md
│   │   └── A2.md
│   ├── content_B/          # 内容分类 B
│   │   ├── B1.md
│   │   └── B2.md
│   ├── content_C/          # 内容分类 C
│   │   └── C1.md
│   ├── public/             # 静态文件（图片、logo 等）
│   └── .vitepress/         # VitePress 配置（重要！）
│       └── config.mts      # 站点配置文件
├── package.json            # 项目依赖和脚本
└── README.md               # 这个文件！
```
