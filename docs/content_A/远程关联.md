# 远程关联同步项目手册

## 查看状态

## 一、查看本地项目关联的远程仓库

使用 `git remote -v` 命令
在本地项目目录下打开终端（Git Bash 或命令行），运行以下命令：

```sh
git remote -v
```

## 二、更换远程仓库地址

方法：直接修改远程仓库 URL（推荐）
如果只是想更换远程仓库的地址（比如从 Gitee 换成 GitHub），可以使用 `git remote set-url` 命令：

```sh
# 查看当前的远程仓库
git remote -v

# 修改远程仓库地址（将 origin 的 URL 改为新地址）
git remote set-url origin <新的仓库URL>

# 修改远程仓库地址（SSH Gitee）
git remote set-url git@gitee.com:shub77/vitepress-tip.git

# 再次查看确认
git remote -v
```

**例如**，将 origin 从 Gitee 改为 GitHub：

```sh
git remote set-url origin https://github.com/shub2026/vitepress-tip.git
```

**总结**
|操作 |命令|
| --- | --- |
查看关联| git remote -v
修改 URL| git remote set-url origin <新地址>
删除关联| git remote remove origin
添加关联| git remote add origin <地址>

## 同步切换

## 一、上传代码

#### 方式一：HTTPS方式（新手推荐，较简单）

进入项目文件夹，右键 → 「Git Bash Here」：

```sh
# 1. 初始化本地仓库
git init

# 2. 添加所有文件到暂存区
git add .

# 3. 提交到本地仓库
git commit -m "首次提交：上传项目文件"

# 4. 关联远程仓库
git remote add origin https://gitee.com/你的用户名/仓库名.git

# 5. 推送到远程仓库
git push -u origin main   # 如果默认分支是main，则写 origin main
```

#### 方式二：SSH方式（配置后免密，长期推荐）

```sh
# 1. 初始化本地仓库
git init

# 2. 添加所有文件到暂存区
git add .

# 3. 提交到本地仓库
git commit -m "首次提交：上传项目文件"

# 第4步关联远程仓库 使用SSH地址
git remote add origin git@gitee.com:shub77/vitepress-tip.git

# 第5步推送（配置SSH后无需密码）
git push -u origin main
```

## 二、初次拉取代码

1. 在任意工作目录`CMD`
2. 克隆远程仓库

```sh
# SSH方式
git clone git@gitee.com:shub77/vitepress-tip.git

# HTTPS方式
git clone https://gitee.com/shub77/vitepress-tip.git
```

## 三、同步操作流程（核心）

_标准工作流程_

```sh
# 1. 每次开始编码前，先拉取最新代码（非常重要！）
git pull origin main

# 2. 修改代码...

# 3. 保存后，将修改提交并推送
git add .
git commit -m "本次修改的说明"
git push origin main
```

---

:::tip 高效切换建议

- 切换电脑前：在当前电脑执行git add → git commit → git push，确保远程仓库是最新状态
- 换到另一台电脑：先执行git pull origin main拉取最新代码，再开始开发
- 避免同时修改同一文件：如果两台电脑同时修改了同一个文件的同一行，拉取时会触发冲突，需要手动解决
