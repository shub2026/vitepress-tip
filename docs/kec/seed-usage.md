# 种子数据使用指南

## 安全特性

重构后的种子脚本具有以下安全保障：

### ✅ 保护超级管理员账号
- **智能检查**：只在 `admin` 用户不存在时才创建
- **防止覆盖**：已存在的账号（包括修改后的密码）不会被重置
- **生产安全**：多次执行不会丢失已有管理员账号

### ✅ 环境保护机制
- **生产环境默认保护**：不会自动清空业务数据
- **开发环境友好**：可选择性清空数据用于测试
- **强制重置选项**：提供明确的开关控制

---

## 使用场景

### 场景1：首次部署（推荐）

```bash
cd server
npm run db:seed
```

**效果**：
- ✓ 创建超级管理员账号（如果不存在）
- ✓ 保留所有现有业务数据
- ✓ 安全可重复执行

---

### 场景2：开发环境重置数据

```bash
# 方式1：设置开发模式
NODE_ENV=development npm run db:seed

# 方式2：显式启用开发种子
DEV_SEEDS=true npm run db:seed
```

**效果**：
- ⚠️ 清空所有现有数据
- ✓ 重新创建超级管理员
- ✓ 创建开发测试数据（待实现）

---

### 场景3：完全重置（危险操作）

```bash
FORCE_RESET=true npm run db:seed
```

**效果**：
- ⚠️⚠️ **清空所有数据（不可恢复）**
- ✓ 重新创建超级管理员
- ⚠️ 仅用于测试环境或灾难恢复

---

## 环境变量说明

| 变量 | 值 | 作用 |
|------|-----|------|
| `NODE_ENV` | `production` / `development` | 自动判断环境类型 |
| `FORCE_RESET` | `true` / `false` | 强制清空所有数据 |
| `DEV_SEEDS` | `true` / `false` | 启用开发测试数据 |

---

## 各环境行为对比

| 操作 | 开发环境 (默认) | 生产环境 (默认) | 强制重置 |
|------|----------------|----------------|---------|
| 清空数据 | ✓ 是 | ✗ 否 | ✓ 是 |
| 创建 admin | 仅当不存在 | 仅当不存在 | 仅当不存在 |
| 创建测试数据 | ✓ 是（待实现） | ✗ 否 | ✗ 否 |
| 安全性 | 中等 | 高 | 低 |

## 常见问题

### Q1: 生产环境执行种子脚本会删除我的数据吗？
**A**: 不会。默认情况下，生产环境只会检查并创建 admin 账号（如果不存在），不会清空任何业务数据。

### Q2: 我忘记了 admin 密码怎么办？
**A**: 不要重新执行种子脚本！应该：
1. 直接修改数据库中的密码哈希值
2. 或者使用 FORCE_RESET（会清空所有数据，慎用！）

### Q3: 如何添加更多测试数据？
**A**: 编辑 `seed.js` 中的 `createDevSeeds()` 函数，添加你的测试数据创建逻辑。

### Q4: 可以创建多个管理员账号吗？
**A**: 当前版本只创建一个 `admin` 账号。如需多账号，请通过系统界面创建，或修改种子脚本。

## 最佳实践

### ✅ 推荐做法
1. **首次部署**：直接运行 `npm run db:seed`
2. **日常维护**：不要重复执行种子脚本
3. **密码管理**：首次登录后立即修改默认密码
4. **备份策略**：定期备份数据库，而不是依赖种子脚本

### ❌ 避免操作
1. **不要在生产环境使用 `FORCE_RESET`**
2. **不要频繁执行种子脚本**
3. **不要将自定义业务数据放入种子脚本**
4. **不要忽略密码修改警告**

## 迁移到生产环境的步骤

```bash
# 1. 配置生产数据库
vim .env
# 修改 DATABASE_URL 为 MySQL 连接字符串

# 2. 执行数据库迁移
npx prisma migrate deploy

# 3. 生成 Prisma Client
npx prisma generate

# 4. 初始化超级管理员（安全，可重复执行）
npm run db:seed

# 5. 启动应用
npm start
```

## 故障排查

### 问题：提示 "admin 已存在" 但无法登录
**解决**：种子脚本不会重置密码。需要通过数据库工具手动重置：

```sql
UPDATE users SET password = '新的bcrypt哈希' WHERE username = 'admin';
```

### 问题：执行后没有任何数据
**解决**：检查是否误用了 `FORCE_RESET=true`。如果是，需要重新导入业务数据。

### 问题：想要恢复测试数据
**解决**：种子脚本目前不包含详细测试数据。请使用系统的导入功能或手动创建。
